# Shop Bot

Telegram-бот для автопродажи ключей подписок с каталогом, личным кабинетом, реферальной программой и Telegram-админкой.

## Что умеет

- каталог категорий и товаров;
- карточки товаров с остатками;
- покупка с внутреннего баланса;
- оплата и пополнение баланса через CryptoBot и Lolzteam Market;
- история заказов и выдача купленных ключей;
- реферальная программа с начислением процента;
- Telegram-админка для управления категориями, товарами, складом, пользователями и настройками;
- сидирование стартового ассортимента.

## Быстрый старт

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python3 main.py
```

## Запуск через Docker Compose + Caddy

```bash
cp .env.example .env
docker compose up --build -d
```

- поднимаются сразу два сервиса: `bot` и `caddy`;
- контейнеры читают переменные из `.env`;
- SQLite хранится в локальной папке `data/`, она примонтирована в контейнер;
- Caddy автоматически получает и продлевает HTTPS-сертификаты для `shop.vexnd.com`;
- наружу публикуются только `80` и `443`, а сам Python-сервис остается во внутренней Docker-сети;
- webhook, страница оплаты и web-админка будут доступны через `https://shop.vexnd.com`.

Перед первым запуском на сервере:

- направьте `A`-запись домена `shop.vexnd.com` на IP сервера;
- откройте входящие порты `80` и `443`;
- убедитесь, что `PUBLIC_BASE_URL=https://shop.vexnd.com` указан в `.env`.

Остановить сервис:

```bash
docker compose down
```

## Платежи

- `CryptoBot`:
  - получите `CRYPTOPAY_API_TOKEN` в `@CryptoBot`;
  - webhook: `POST /payments/cryptobot/webhook/<CRYPTOPAY_WEBHOOK_PATH_TOKEN>`;
  - success URL: `GET /payments/cryptobot/success`;
  - подпись проверяется через `crypto-pay-api-signature`, старые события отбрасываются по `request_date`, а paid-webhook дополнительно перепроверяется запросом `getInvoices/getInvoice` перед зачислением.
- `Lolzteam Market`:
  - задайте `LOLZ_MERCHANT_ID`, `LOLZ_API_TOKEN` и `LOLZ_MERCHANT_TOKEN`;
  - webhook: `POST /payments/lolzteam/callback/<LOLZ_WEBHOOK_PATH_TOKEN>`;
  - success URL: `GET /payments/lolzteam/success`;
  - API-запросы идут с `Authorization: Bearer <LOLZ_API_TOKEN>`, а webhook проверяется по `x-secret-key`, который должен совпадать с merchant token.
  - после webhook invoice повторно читается из `GET /invoice` перед подтверждением в боте.
- Для включённых платёжных провайдеров обязателен `PUBLIC_BASE_URL`, иначе бот не запустится: без него нельзя корректно собрать callback/success URLs и webhook-маршруты.
- Для обоих провайдеров доступны ручная проверка статуса в боте и фоновая синхронизация pending-платежей.

## Telegram-админка

- Добавьте свой Telegram ID в `ADMIN_IDS`.
- Откройте `/admin` в боте.
- Через меню можно:
  - добавлять и отключать категории;
  - добавлять и редактировать товары;
  - загружать ключи пачкой, по одному ключу на строку;
  - искать пользователя по Telegram ID и пополнять баланс;
  - менять ссылку поддержки и процент реферальной награды.

## Web-админка

- Web-админка доступна по адресу `https://shop.vexnd.com/admin` на сервере или по `PUBLIC_BASE_URL/admin`.
- Она запускается в том же процессе и контейнере, что и бот и платежные webhook routes.
- Для входа задайте отдельные переменные:
  - `ADMIN_WEB_USERNAME`
  - `ADMIN_WEB_PASSWORD`
  - `ADMIN_WEB_SECRET`
  - `ADMIN_WEB_SESSION_TTL_HOURS`
- Внутри панели доступны:
  - dashboard со статистикой, low-stock сигналами, последними заказами и платежами;
  - управление категориями и товарами;
  - пополнение склада ключами;
  - поиск пользователей и ручное пополнение баланса;
  - просмотр заказов, платежей и базовых настроек магазина.

## Как работает интеграция

- Для оплаты товара через CryptoBot и Lolzteam конкретный ключ резервируется на время жизни счёта, чтобы не было двойной продажи.
- Подтверждение оплаты идёт идемпотентно: повторный webhook или повторная ручная проверка не приведут к двойному зачислению или повторной выдаче.
- После оплаты бот автоматически пополняет баланс или выдаёт товар, а при сбое callback статус можно дотянуть через фоновую синхронизацию.
